# 📊 Настройка статистики просмотров

## Шаг 1: Добавить поле views в таблицу cars

### Вариант A: Через Supabase Dashboard (рекомендуется)

1. **Откройте SQL Editor:**
   https://app.supabase.com/project/dybmmygpbyaawmiweidu/sql/new

2. **Скопируйте и выполните этот SQL:**

```sql
-- Добавление поля views для статистики просмотров
ALTER TABLE cars ADD COLUMN IF NOT EXISTS views INTEGER DEFAULT 0;

-- Создаем индекс для быстрой сортировки по просмотрам
CREATE INDEX IF NOT EXISTS idx_cars_views ON cars(views DESC);

-- Обновляем существующие записи
UPDATE cars SET views = 0 WHERE views IS NULL;
```

3. **Нажмите RUN** (или Ctrl+Enter)

4. **Должно появиться:** `Success. No rows returned`


## Шаг 2: Создать RPC функцию для атомарного инкремента (опционально)

Это позволит увеличивать счетчик безопасно без race conditions.

**В SQL Editor выполните:**

```sql
CREATE OR REPLACE FUNCTION increment_car_views(car_id bigint)
RETURNS integer AS $$
DECLARE
  new_views integer;
BEGIN
  UPDATE cars
  SET views = COALESCE(views, 0) + 1
  WHERE id = car_id
  RETURNING views INTO new_views;

  RETURN new_views;
END;
$$ LANGUAGE plpgsql;
```

## Как работает статистика

### Автоматический подсчет

Каждый раз, когда пользователь открывает страницу `/catalog/[id]`, счетчик автоматически увеличивается на 1.

**Код в `src/app/catalog/[id]/page.tsx`:**

```typescript
// Увеличиваем счетчик просмотров
fetch(`/api/cars/views?id=${id}`, { method: 'POST' })
```

### API endpoint

**`POST /api/cars/views?id=123`** - увеличивает счетчик на 1

**Ответ:**
```json
{
  "views": 42
}
```
