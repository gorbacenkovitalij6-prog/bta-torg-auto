# 📊 Графики статистики просмотров готовы!

## ✅ Что добавлено

Интерактивные графики статистики просмотров по дням и неделям с возможностью переключения периодов.

---

## 🎯 Основные функции

### 1. Интерактивные графики

- **📈 Линейный график** - динамика просмотров во времени
- **📊 Столбчатый график** - распределение просмотров по периодам
- **🔄 Переключение режимов** - "По дням" и "По неделям"
- **📅 Выбор периода** - 7, 30 или 90 дней

### 2. Карточки статистики

- **Всего просмотров** за выбранный период
- **Среднее в день** - автоматический расчет
- **Максимум в день** - пиковый показатель

### 3. Автоматический сбор данных

- **Таблица `daily_views`** - хранит статистику по дням
- **Автоматическая запись** при каждом просмотре
- **RPC функции** для быстрой выборки данных

## 🗄️ База данных

### Новая таблица `daily_views`

```sql
CREATE TABLE daily_views (
  id BIGSERIAL PRIMARY KEY,
  car_id BIGINT REFERENCES cars(id) ON DELETE CASCADE,
  view_date DATE NOT NULL DEFAULT CURRENT_DATE,
  views_count INTEGER DEFAULT 1,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  UNIQUE(car_id, view_date)
);
```

**Что она хранит:**
- `car_id` - ID автомобиля
- `view_date` - дата просмотра
- `views_count` - количество просмотров за день
- Уникальное ограничение на пару (car_id, view_date)

### RPC Функции

**1. `get_daily_stats(days_count)` - общая статистика**

```sql
SELECT * FROM get_daily_stats(30); -- Последние 30 дней
```

Возвращает:
```
view_date  | total_views
-----------+-------------
2025-02-01 | 45
2025-02-02 | 67
2025-02-03 | 52
```

**2. `get_car_daily_stats(car_id, days_count)` - статистика авто**

```sql
SELECT * FROM get_car_daily_stats(1, 30); -- Авто #1, последние 30 дней
```

Возвращает:
```
view_date  | views_count
-----------+-------------
2025-02-01 | 12
2025-02-02 | 18
2025-02-03 | 9
```

**3. Обновленная `increment_car_views(car_id)`**

Теперь функция:
1. Увеличивает общий счетчик в `cars.views`
2. Записывает или обновляет счетчик в `daily_views`

## 📊 Как это работает

### Запись статистики

Когда пользователь открывает страницу `/catalog/[id]`:

```typescript
// 1. Фронтенд отправляет запрос
fetch(`/api/cars/views?id=${id}`, { method: 'POST' })

// 2. API вызывает RPC функцию
supabase.rpc('increment_car_views', { car_id: id })

// 3. PostgreSQL функция:
//    - Увеличивает cars.views на 1
//    - Добавляет/обновляет запись в daily_views
```

### Отображение графиков

В админ-панели:

```typescript
// 1. Запрос статистики
fetch('/api/stats?days=30')

// 2. API вызывает RPC или делает прямую выборку
get_daily_stats(30)

// 3. Компонент StatisticsCharts:
//    - Рендерит линейный график (динамика)
//    - Рендерит столбчатый график (распределение)
//    - Группирует по неделям (опционально)
```

## 🎨 Интерфейс

### Переключатели

**Режим просмотра:**

- Tooltip с точными значениями

**Столбчатый график:**
- Зеленые столбцы
- Показывает распределение
- Плавные закругленные углы

## 🔧 API Endpoint

### GET /api/stats

**Параметры:**
- `days` - количество дней (7, 30, 90)
- `carId` - ID автомобиля (опционально)

**Примеры:**

```bash
# Общая статистика за 30 дней
GET /api/stats?days=30

# Статистика конкретного авто за 7 дней
GET /api/stats?days=7&carId=1
```

**Ответ:**

```json
[
  {
    "view_date": "2025-02-01",
    "total_views": 45
  },
  {
    "view_date": "2025-02-02",
    "total_views": 67
  }
]
```
