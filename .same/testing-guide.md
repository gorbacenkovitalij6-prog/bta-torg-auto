## Шаг 1: Настройка Supabase (если еще не сделано)

### 1.1 Откройте Supabase Dashboard

Перейдите на: https://app.supabase.com/project/dybmmygpbyaawiweidu

### 1.2 Создайте таблицу `cars`

1. **SQL Editor** (в левом меню)
2. **New query**
3. Скопируйте весь SQL из файла `brabus-clone/.same/setup-database.sql`
4. **Run** (или Ctrl+Enter)
5. Должно появиться: `Success. No rows returned`

**Проверка:** В Table Editor должна появиться таблица `cars` с полями:
- `id` (BIGSERIAL)
- `title` (TEXT)
- `subtitle` (TEXT)
- `images` (JSONB) ← **Важно для множественной загрузки**
- `price`, `brand`, `model`, `year`, `mileage`, `description`
- `createdAt` (TIMESTAMP)

### 1.3 Создайте Storage bucket

1. **Storage** (в левом меню)
2. **Create bucket**
3. Имя: `car-images`
4. **Public bucket**: ✅ **Обязательно включить!**
5. **Create**

### 1.4 Настройте Storage политики

В **Storage** → `car-images` → **Policies**:

**Если политик нет, создайте:**

```sql
-- Политика 1: Публичное чтение
CREATE POLICY "Публичное чтение изображений"
  ON storage.objects FOR SELECT
  USING (bucket_id = 'car-images');

-- Политика 2: Разрешить загрузку
CREATE POLICY "Разрешить загрузку изображений"
  ON storage.objects FOR INSERT
  WITH CHECK (bucket_id = 'car-images');
```

**Проверка:** Должно быть минимум 2 политики для bucket `car-images`.

## 🐛 Решение проблем

### ❌ Ошибка: "Ошибка загрузки: Upload failed"

**Проверьте:**

1. **Консоль браузера (F12)**:
   - Ищите красный текст с ошибкой
   - Скопируйте текст ошибки

2. **Терминал dev сервера**:
   - Ищите строки с `ERROR` или `Supabase upload error`
   - Проверьте сообщение об ошибке

**Возможные причины:**

| Ошибка | Решение |
|--------|---------|
| `Bucket not found` | Создайте bucket `car-images` в Supabase Storage |
| `Supabase not configured` | Проверьте `.env` файл, должен быть `SUPABASE_SERVICE_ROLE_KEY` |
| `Policy violation` | Проверьте политики Storage (должна быть INSERT policy) |
| `File size too large` | Уменьшите размер изображений (макс 5 МБ по умолчанию) |

### ❌ Ошибка: "Не удалось сохранить объявление"

**Проверьте:**

1. Таблица `cars` создана в Supabase
2. RLS политики настроены (см. `.same/setup-database.sql`)
3. Консоль браузера и терминал на наличие ошибок

### ❌ Фото загрузились, но не отображаются в карточке

**Проверьте:**

1. **next.config.js** → должен содержать:
   ```js
   {
     protocol: 'https',
     hostname: 'dybmmygpbyaawiweidu.supabase.co',
   }
   ```

2. **Перезапустите dev сервер**:
   ```bash
   # В терминале нажмите Ctrl+C
   bun run dev
   ```

3. **Bucket настроен как публичный** (Storage → car-images → Settings)

### ❌ Стрелки галереи не появляются

**Проверьте:**

1. Загружено ли больше 1 фото (стрелки появляются только если `images.length > 1`)
2. Наведите курсор на карточку (стрелки появляются при hover)
3. Проверьте консоль браузера на ошибки JavaScript
